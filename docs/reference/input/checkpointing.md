# Checkpointing models

## CheckpointModelSpec { #checkpointmodelspec }

Specification of the checkpoint model used by the Data Center. When using checkpointing the system periodically snapshots the progress of a task. When a failure occurs the task resumes from its most recent checkpoint instead of restarting from the beginning.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `checkpointInterval` | Long | no | The wall-clock time between two consecutive checkpoints, in milliseconds. Must be positive. Default value is 1 hour. |
| `checkpointDuration` | Long | no | The time it takes to create a single checkpoint, in milliseconds. Must be positive. Default value is 5 minutes. |
| `checkpointIntervalScaling` | Double | no | Factor by which `checkpointInterval` is scaled after each checkpoint. A value of `1.0` keeps the interval constant. Must be positive. |
